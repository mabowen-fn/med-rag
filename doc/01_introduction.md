# 一、绪论

## 1.1 项目背景和意义

随着人工智能技术的快速发展，大语言模型（Large Language Models, LLMs）在自然语言处理领域取得了突破性进展。然而，通用大语言模型在医疗等垂直行业应用中面临严峻挑战。医疗领域对信息的准确性、可靠性和可追溯性要求极高，任何错误或幻觉都可能导致严重后果。传统的大语言模型存在知识更新滞后、容易产生幻觉、缺乏引用溯源等核心痛点，这些问题在医疗场景中尤为突出。

检索增强生成（Retrieval-Augmented Generation, RAG）技术为解决上述问题提供了有效途径。RAG通过在生成回答前先检索相关知识，将检索到的上下文作为生成依据，从而显著提高回答的准确性和时效性。本项目针对医疗垂直行业，设计并实现了一个基于RAG技术的医疗问答系统，旨在解决医疗领域的知识获取难题。

本项目的核心意义在于：第一，通过混合检索策略（BM25关键词检索+向量语义检索）提高医疗知识检索的准确性和全面性；第二，引入重排序模型（Cross-Encoder Reranker）优化检索结果质量；第三，实现置信度评估和引用溯源机制，增强系统的可信度和透明度；第四，采用本地部署的大语言模型（Qwen3:8b），确保数据隐私和安全。这些技术特性使系统能够为医疗从业者、患者和医学学生提供可靠、可追溯的医疗知识服务。

## 1.2 相关技术介绍

### 1.2.1 检索增强生成（RAG）

检索增强生成是一种结合信息检索和文本生成的技术框架。其核心思想是在大语言模型生成回答之前，先从外部知识库中检索相关文档，将检索结果作为上下文输入给模型，从而引导模型基于真实知识生成回答。RAG技术有效缓解了大语言模型的知识幻觉问题，同时支持知识的动态更新，无需重新训练模型。

### 1.2.2 向量嵌入与语义检索

向量嵌入（Vector Embedding）技术将文本转换为高维向量表示，使得语义相似的文本在向量空间中距离更近。本项目采用BAAI/bge-m3嵌入模型，该模型支持多语言、多粒度，在医疗文本语义理解方面表现优异。基于向量嵌入，系统使用FAISS（Facebook AI Similarity Search）构建高效的向量索引，实现快速的语义相似度检索。

### 1.2.3 BM25关键词检索

BM25是经典的信息检索算法，基于词频（TF）和逆文档频率（IDF）计算查询与文档的相关性。与语义检索相比，BM25在精确匹配医学术语、药品名称、疾病代码等方面具有优势。本项目采用BM25Okapi实现，通过加权融合策略将BM25检索结果与向量检索结果结合，实现混合检索。

### 1.2.4 交叉编码器重排序

重排序（Reranking）是检索后的精排阶段，通过更复杂的模型对候选文档进行重新评分。本项目使用BAAI/bge-reranker-v2-m3交叉编码器模型，该模型同时编码查询和文档，捕捉它们之间的细粒度语义关系，相比双编码器模型具有更高的准确性。重排序显著提升了检索结果的质量，特别是在医疗专业领域。

### 1.2.5 大语言模型

本项目采用Qwen3:8b作为生成模型，通过Ollama框架进行本地部署。Qwen3是阿里云通义千问系列模型，8b版本在保持良好性能的同时，对硬件要求相对较低，适合在消费级GPU上运行。本地部署确保了医疗数据的隐私安全，避免了数据外泄风险。

## 1.3 项目内容与目标

本项目旨在构建一个面向医疗垂直行业的智能问答系统，主要完成以下工作：

**需求分析**：明确医疗问答场景的核心需求，包括准确性、可靠性、可追溯性、响应速度等性能指标，分析医疗从业者、患者和医学学生等不同用户群体的使用需求。

**系统设计**：设计完整的RAG系统架构，包括数据处理层、知识表示层、检索层、重排序层、生成层和用户界面层。重点设计混合检索策略、置信度评估机制、引用溯源系统和多轮对话管理。

**核心实现**：实现基于BM25和向量检索的混合检索器，集成交叉编码器重排序模型，开发置信度评分和路由机制，构建引用追踪和验证系统，实现多轮对话管理。

**优化与创新**：引入动态分块策略优化文档切分，实现基于置信度的安全路由机制，开发引用溯源和验证功能，探索自反思RAG（Self-RAG）等先进技术。

**实验评估**：建立完整的评估体系，包括检索准确率（Recall@K、Precision@K）、生成质量（BLEU、ROUGE-L、BERTScore）、幻觉率、响应时间等指标，通过对比实验验证各技术组件的有效性。

项目目标是构建一个可运行、可复现、可评估的医疗RAG系统，为垂直行业的AI应用提供技术参考和实践经验。
