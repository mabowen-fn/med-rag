# 二、需求分析

## 2.1 功能需求

本系统面向医疗垂直行业，核心功能是提供基于检索增强生成的医疗问答服务。系统需要接收用户的自然语言医疗问题，从医疗知识库中检索相关信息，结合检索到的上下文生成准确、可靠的回答，并提供完整的引用溯源和置信度评估。

在数据处理方面，系统需要支持医疗数据集的加载和预处理。系统采用Huatuo-26M和CMeQA等医疗问答数据集作为知识来源，需要实现数据集的自动化加载、清洗和格式化。数据需要经过动态分块处理，将长文档切分为适合检索和生成的文本块，同时保持语义完整性。分块策略需要支持可配置的块大小和重叠区域，以平衡检索精度和上下文连贯性。

在知识表示方面，系统需要构建双索引结构。一方面，使用BGE-M3嵌入模型将文本块转换为1024维向量表示，构建FAISS向量索引以支持语义检索；另一方面，构建BM25关键词索引以支持精确匹配。双索引结构能够同时捕捉语义相似性和关键词匹配，提高检索的全面性和准确性。

在检索流程方面，系统需要实现混合检索策略。当用户提出问题时，系统同时执行向量检索和BM25检索，然后通过加权融合策略合并两路检索结果。向量检索权重设置为0.7，BM25检索权重设置为0.3，这一配置基于医疗领域特点进行了优化。检索完成后，系统使用BGE-Reranker-v2-m3交叉编码器对候选文档进行重排序，进一步提升检索质量。

在生成流程方面，系统需要实现基于置信度的安全路由机制。系统计算检索结果的置信度分数，根据置信度水平决定后续处理策略。当置信度高于0.7时，系统直接生成回答；当置信度在0.5到0.7之间时，系统生成回答但附加警告信息；当置信度低于0.5时，系统拒绝生成回答并建议用户咨询专业医疗人员。这一机制确保了系统在高不确定性场景下的安全性。

在引用溯源方面，系统需要实现完整的引用追踪机制。系统记录每个检索文档的来源、相关性和得分信息，在生成回答时将引用信息传递给用户。用户可以通过引用编号追溯到原始文档，验证回答的准确性和可靠性。引用信息包括文档来源、检索得分和相关性说明，帮助用户评估信息的可信度。

在用户交互方面，系统需要提供Web界面支持多轮对话。用户可以连续提问，系统维护对话历史并在生成回答时考虑上下文信息。界面需要展示回答内容、置信度分数、引用列表和系统状态，提供直观的交互体验。系统还需要支持配置调整，允许用户切换不同的处理模式（基础模式或SOTA模式）和调整检索参数。

在评估方面，系统需要实现全面的性能评估功能。系统支持多种评估指标，包括检索准确率（Recall@K、Precision@K、MRR）、生成质量（BLEU、ROUGE-L、BERTScore）、幻觉率和响应时间。系统能够自动生成评估报告，对比不同配置和模式的性能差异，为系统优化提供数据支持。

## 2.2 非功能需求

在性能方面，系统需要满足实时交互的要求。端到端响应时间应控制在10秒以内，其中检索阶段不超过2秒，生成阶段不超过8秒。系统需要支持至少10,000个文档的知识库规模，在此规模下检索延迟保持在可接受范围内。系统需要支持CPU和GPU两种运行模式，在GPU加速下性能应显著提升。

在准确性方面，系统需要将幻觉率控制在30%以下，相比基线LLM的80%幻觉率实现显著降低。检索准确率（Recall@10）应达到85%以上，确保相关信息能够被有效检索。生成回答的忠实度（Faithfulness）应达到90%以上，确保回答内容与检索到的上下文保持一致。

在可靠性方面，系统需要具备完善的错误处理机制。当组件加载失败时，系统应提供降级方案而非直接崩溃。当检索结果质量不佳时，系统应通过置信度机制识别并拒绝生成低质量回答。系统需要记录详细的日志信息，便于问题诊断和性能分析。

在可扩展性方面，系统需要采用模块化设计，各组件之间通过清晰的接口进行交互。这种设计使得系统能够方便地替换或升级特定组件，例如更换嵌入模型、调整检索策略或集成新的评估指标。系统配置需要集中管理，支持通过配置文件调整参数而无需修改代码。

在部署方面，系统需要支持多种硬件环境。在开发环境中，系统应能在Mac M1等CPU设备上运行，便于开发和测试。在生产环境中，系统应能在NVIDIA RTX 4090等GPU设备上运行，充分利用GPU加速提升性能。系统需要提供自动化部署脚本，简化环境配置和依赖安装过程。

在安全性方面，系统采用本地部署策略，所有数据处理和模型推理均在本地完成，不涉及外部API调用，确保医疗数据的隐私安全。系统通过置信度路由机制避免在低置信度场景下生成可能错误的回答，降低医疗风险。

## 2.3 性能指标

系统性能评估涵盖多个维度。检索性能指标包括Recall@K（检索到的相关文档比例）、Precision@K（检索结果中相关文档比例）和MRR（第一个相关文档的排名倒数），这些指标衡量检索系统的有效性。

生成质量指标包括BLEU（基于n-gram重叠的相似度）、ROUGE-L（基于最长公共子序列的相似度）和BERTScore（基于语义相似度的评分），这些指标衡量生成回答与参考答案的匹配程度。

幻觉率是医疗问答系统的关键指标，衡量生成内容中无法从检索上下文得到支持的比例。系统目标是将幻觉率从基线的80%降低到30%以下，通过检索增强和引用验证显著减少虚假信息的生成。

响应时间指标衡量系统的实时性能，包括检索时间、重排序时间、生成时间和端到端总时间。系统需要在保证质量的前提下尽可能降低延迟，提供流畅的用户体验。

置信度分数反映系统对回答可靠性的评估，基于检索质量、上下文完整性和生成一致性等多个因素综合计算。置信度不仅用于安全路由，也作为性能评估的重要指标，高置信度回答应具有更高的准确性。
